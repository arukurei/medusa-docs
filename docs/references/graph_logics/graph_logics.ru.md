# GraphLogic

Система **GraphLogic** — это мозг вашего графа. Она позволяет автоматизировать поведение сотен узлов, не прописывая условия для каждого индивидуально. Ниже приведен детальный разбор того, как создавать свои правила, как работают приоритеты и какие нюансы стоит учитывать.

Любой скрипт логики, наследуемый от `GraphLogic`, работает как фильтр в конвейере. Когда Граф обновляется, он прогоняет каждый Атом через этот конвейер правил.

## 1. Метод `get_permission`: Кто имеет право?
Этот метод отвечает только на один вопрос: «Какое мнение у этого правила относительно этого Атома?». Он не меняет состояние Атома, а возвращает значение типа `Return`:

*   **`Return.BLOCK` (Приоритет 1):** Жесткое вето. Если хотя бы одно правило возвращает `BLOCK`, атом никогда не станет доступным, даже если другие правила «ЗА». Используйте это для условий «взаимоисключающих талантов» или сюжетных ограничений.
*   **`Return.ALLOW` (Приоритет 2):** Разрешение. Правило говорит: «Я считаю, что этот атом можно открыть». Чтобы атом стал `AVAILABLE` (доступным), нужно хотя бы одно разрешение при отсутствии запретов.
*   **`Return.PASS` (Приоритет 3):** Нейтралитет. Правило говорит: «Мне всё равно, пусть решают другие правила». Это значение по умолчанию.

!!!example
    Если на графе нет ни одного правила, возвращающего `ALLOW`, то все атомы останутся в `LOCKED`, так как никто не дал явного разрешения.

## 2. Метод `apply_status`: Управление визуальным состоянием

В то время как `get_permission` просто возвращает статус, метод `apply_status` — это место, где происходит реальное изменение свойств Атома (`status`, `modulate`, `visible` и т.д.).

**Пример расширенной логики:**
```gdscript
@tool
class_name HiddenFogLogic extends GraphLogic

func apply_status(atom: Atom, graph: Graph, progress: ProgressDB = null) -> bool:
	# Если это навык "Скрытый", и у него нет открытых соседей
	if atom.tags.has(&"secret") and not graph.has_unlocked_neighbor(atom, progress):
		atom.status = Atom.Status.HIDDEN
		atom.visible = false # Физика всё еще работает, но мы его не видим
		return false # Прерываем цепочку, чтобы другие правила не сделали его LOCKED или AVAILABLE
	
	atom.visible = true # Показываем, если условия не сработали
	return true # Позволяем следующим правилам (например, ProgressLogic) уточнить статус
```

!!! warning
    **Порядок имеет значение:** Правила в массиве `graph_logics` выполняются сверху вниз.

!!! note
    Метод возвращает `bool`. 
        Если вернуть `false`, Граф поймет, что это правило было критическим, и **прекратит** опрос следующих правил для этого атома. 
        Если вернуть `true`, Граф продолжит опрос следующего правила в списке.

## 3. Нюансы и хитрости

#### Инспекция тегов (All Tags)
Используйте метод `atom.get_all_tags()`. Он объединяет теги, прописанные вручную в ноде `Atom`, и теги, которые лежат в ресурсе `AtomData`. Это позволяет создавать правила, работающие как для конкретных иконок, так и для целых классов ресурсов.

#### Использование ProgressDB в логике
Ваша логика имеет прямой доступ к `progress`. Вы можете проверять не только факт покупки, но и уровень:

```gdscript
# Внутри get_permission
if progress.get_level(&"base_intelligence") < 5:
    return Return.BLOCK # Мало интеллекта для этого навыка
```

#### Метод `graph.has_unlocked_neighbor()`
Это «золотой метод» для деревьев навыков. Он автоматически проверяет всех родителей (или детей, если указать `ConnectionDir.OUTBOUND`) и говорит, разблокирован ли хотя бы один из них в базе прогресса.

## Порядок правил в Графе (Best Practice)
Для стабильной работы дерева навыков располагайте логику в `graph_logics` так:

1.  **BlockLogics** (Запрещаем всё, что нельзя при любых условиях).
2.  **SpecialRules** (Например, `TagLogic` для корней дерева).
3.  **ConnectionLogic** (Проверка связей: можно ли купить следующий навык?).
4.  **ProgressLogic** (Финальная проверка: куплен ли этот навык уже?).

Этот порядок гарантирует, что купленный навык всегда останется в статусе `UNLOCKED`, но если связь с ним еще не дошла, он будет `LOCKED`, а если его заблокировал сюжетный `BlockLogic`, он станет `DISABLED`.
